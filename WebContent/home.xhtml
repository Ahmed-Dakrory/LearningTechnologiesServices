Welcome

This is an example of a simple Facelets template. Here, header, footer and side menu bar appear from the template file (content.xhtml). This section (rendering here--what you're reading) appears from the template client (home.xhtml).

All (header, footer and content) are united in the template file, content.xhtml, which is summarily included by this file (home.html).

How this works...

The files header.xhtml and footer.xhtml are NOT anything to do with templating, but with mere inclusion. The template file happens to include them.

content.xhtml, the template file, acts with this file (home.xhtml), also formally known as the template client or "consumer," to accomplish the templating in our example.

Think of a single page (this example). The template file is the one that makes use of while the template client makes use of . home.xhtml and not content.xhml is what is "visible" to the browser user (i.e.: is on the URI) who does not request the template file (content.xhml), header.xhtml, or footer.xhtml, but only this file (home.xhtml). That's why all except this file are subsumed under the template subdirectory.
